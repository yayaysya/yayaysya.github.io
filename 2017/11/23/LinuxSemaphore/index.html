信号量的概念与作用

当用户创立多个线程／进程时，如果不同线程／进程同时读写相同的内容，则可能造成读写错误，或者数据不一致。
此时，需要通过加锁的方式，控制核心区域(critical section)的访问权限。

对于semaphore而言，在初始化变量的时候可以控制允许多少个线程／进程同时访问一个critical section，其他的线程／进程会被堵塞，直到有人解锁。

pv操作

信号量是一个确定的二元组（s，q)
其中

  • s是一个具有非负初值的整形变量;
  • q是一个初始状态为空的队列;

整形变量s表示系统中某类资源的数目：

  • 当 S ≥ 0 时，表示系统中当前可用资源的数目
  • 当 S < 0 时，其绝对值表示系统中因请求该类资源而被阻塞的进程数目

运作方式：

  1. 初始化，给与它一个非负数的整数值。

  2. 企图进入临界区块的进程，需要先运行 P（wait()）。
    运行 P（wait()），信号量S的值将被减少。

  • 减少后, 当信号量S<0时，进程会被挡住，不能继续；
  • 减少后, 当信号量S>=0时，进程可以获准进入临界区块。
  1. 结束离开临界区块的进程，将会运行 V(又称signal())。
    运行 V（又称signal()），信号量S的值会被增加。

当信号量S不为负值时，先前被挡住的其他进程，将可获准进入临界区块

具体的使用

结果

可以看到在同时运行./xinhaoliang o 和 ./xinhaoliang的情况下
X和o分别都是成对出现的
所以可以推断执行的过程

有一个进程先进入临界区域(p操作), 然后另一个进程在等待
等进程退出临界区域(v操作), 另一个进程才进入了临界区域(p操作).
所以在第一个进程在临界区内sleep的时候, 另一个进程就在外面等待

与锁的关系

其实当设置信号量的最大值为1的时候, 就相当于锁的作用
Mutex相当于只允许一个线程／进程访问的semaphore。

另外还有一种读写锁

它允许同时存在多个阅读者(reader)，但任何时候至多只有一个写者(writer)，且不能于读者共存。
就是在锁的基础上放宽条件: 可以允许同时存在多个读的进程

关于死锁以及死锁的条件

死锁是指两个或多个线程／进程之间相互阻塞，以至于任何一个都不能继续运行

线程A占有lock A，并且尝试获取lock B；而线程2占有lock B，尝试获取lock A。此时，两者相互阻塞，都无法继续运行。

死锁的条件

  • 互斥条件(Mutual exclusion)：资源不能被共享，只能由一个进程使用。(
  • 请求与保持条件(Hold and wait)：已经得到资源的进程可以再次申请新的资源。
  • 非剥夺条件(No pre-emption)：已经分配的资源不能从相应的进程中被强制地剥夺。
  • 循环等待条件(Circular wait)：系统中若干进程组成环路，该环路中每个进程都在等待相邻进程正占用的资源。

预防死锁的思想, 死锁的解决办法

1. 忽略该问题。例如鸵鸟算法，该算法可以应用在极少发生死锁的的情况下。为什么叫鸵鸟算法呢，因为传说中鸵鸟看到危险就把头埋在地底下，可能鸵鸟觉得看不到危险也就没危险了吧。跟掩耳盗铃有点像。
2. 检测死锁并且恢复。
3. 仔细地对资源进行动态分配，以避免死锁。
4. 通过破除死锁四个必要条件之一，来防止死锁产生。
